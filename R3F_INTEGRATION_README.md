# 🎨 R3F Storefront Integration для realize-test

Це документація до інтеграції React Three Fiber 3D конфігуратора з Shopify Theme **realize-test** як storefront-збірки.

---

## 📋 Огляд проекту

| Компонент | Статус | Описання |
|-----------|--------|----------|
| **pupupu** | ✅ Розробка | Remix app з R3F, вихідна розробка (admin) |
| **realize-test** | ✅ Тема | Shopify Theme, вбудований storefront configurator |
| **Маршрутизація** | ✅ Зафіксована | `/products/:handle` → R3F якщо має `custom.id` |
| **Бандлування** | ✅ Vite config | `npm run build:theme` компілює в `assets/r3f-*.{js,css}` |
| **Інтеграція** | ✅ Liquid | Snippet `r3f-configurator.liquid` в product page |
| **Cart** | ✅ Properties | Line item properties з конфіга |
| **CI/CD** | ✅ GitHub Actions | Автоматичний push на staging + manual production |

## 🚀 Швидкий старт

### 1. Локальна розробка (паралельний режим)

**Terminal 1: Бандл (watch mode)**
```bash
cd D:\work\configurators\pupupu
npm install
npm run build:theme:watch
```

**Terminal 2: Тема (dev mode)**
```bash
cd D:\work\configurators\realize-test
shopify theme dev --store mystore.myshopify.com
```

Браузер відкриється на `http://localhost:9000`. Покладіть на product з `custom.id` метаполем.

### 2. Тестування

1. Завантажити конфігуратор (очікування ~3-5 сек)
2. Персоналізувати (колір, текст, опції)
3. "Add to Cart" → перевірити properties
4. "/cart" → "/checkout" (нативний Shopify flow)

### 3. Деплой на staging

```bash
git push origin main
# GitHub Actions автоматично:
# 1. Компілює R3F бандл
# 2. Pushає тему на staging
```

### 4. Production (manual)

GitHub Actions → **Deploy Theme to Production** → Run workflow

## 🔧 Налаштування GitHub Actions

Додати secrets в репозиторій:

1. **SHOPIFY_STORE_DOMAIN** = `mystore.myshopify.com`
2. **SHOPIFY_THEME_ACCESS_PASSWORD** = API access token (`shpat_...`)
3. **SHOPIFY_STAGING_THEME_ID** = ID draft theme
4. **SHOPIFY_PRODUCTION_THEME_ID** = ID live theme

Дивись: [CI_AUTOMATION.md](docs/CI_AUTOMATION.md) для детальних інструкцій.

## 🧪 Тестування

### Golden Path
1. Product page з `custom.id` метаполем
2. R3F контейнер завантажується
3. Користувач персоналізує гармент
4. Add to Cart → properties передаються
5. Cart page показує properties
6. Checkout → order creation

### Fallback Cases
- Продукт **без** `custom.id` → нативна product page
- R3F **не завантажується** → fallback текст + нативна UI
- Браузер **без WebGL** → stattic image + variant picker

Дивись: [LOCAL_DEVELOPMENT.md](docs/LOCAL_DEVELOPMENT.md) для детальних тест-кейсів.

## ⚠️ Важливі питання

### 1. Bundle Size
- `r3f-configurator.js` ~ 1.5–3 MB (non-gzipped)
- Розділення, lazy-loading, та CDN для Three.js рекомендується
- Дивись: [RISKS_AND_MITIGATION.md](docs/RISKS_AND_MITIGATION.md#1-розмір-бандла)

### 2. CORS для текстур/моделей
- Переконатись що CDN дозволяє CORS
- Або хостити на тому ж домені
- Дивись: [RISKS_AND_MITIGATION.md](docs/RISKS_AND_MITIGATION.md#2-cors--cdn-доступність-текстурмоделей)

### 3. Контракт даних
- Data-атрибути контейнера мають точно збігатись з очікуванням JS
- Версіонування в `data-contract-version` рекомендується
- Дивись: [R3F_STOREFRONT_ROUTING.md](docs/R3F_STOREFRONT_ROUTING.md#контракт-liquid--r3f-entry)

### 4. Mobile Performance
- GPU detection + quality scaling
- Адаптивна pixelRatio для мобільних
- Дивись: [RISKS_AND_MITIGATION.md](docs/RISKS_AND_MITIGATION.md#8-mobile-performance)

## 🐛 Troubleshooting

| Проблема | Розв'язок |
|----------|----------|
| R3F не завантажується | Перевірити DevTools Console для errors. Дивись [LOCAL_DEVELOPMENT.md](docs/LOCAL_DEVELOPMENT.md#debugging) |
| Bundle файли не знайдені (404) | Запустити `npm run build:theme:watch` в pupupu |
| Cart properties не передаються | Перевірити `initR3FConfigurator()` в browser console |
| GitHub Actions fails | Перевірити secrets та workflow logs |
| Брак памяти / WebGL context limit | Дивись [RISKS_AND_MITIGATION.md](docs/RISKS_AND_MITIGATION.md#5-performance-webgl-context-limit) |
